## 一、产品定位（升级后）

### 旧定位

> AI Agent 驱动的项目展示资产生成器——输入 GitHub 链接，生成讲稿/PPT/One-pager。

### 新定位

> **DemoGen 是一个帮助开发者低成本宣传自己产品的 AI 平台。**
>
> 用户输入产品信息（GitHub 链接、截图、描述），选择目标发布渠道（小红书/Twitter/Product Hunt/GitHub 等），
> DemoGen 自动生成适合该渠道的全套宣传资产——从文案到标注截图到 GIF 到短视频。

### 为什么要升级

旧版只生成讲稿和 PPT，有两个问题：
1. **差异化弱**：和直接用 ChatGPT 写没有本质区别
2. **覆盖面窄**：开发者宣传产品不只需要讲稿，更需要社媒帖子、标注截图、GIF、短视频

调研发现：
- 开发者最大的宣传痛点不是"没有工具做视频"，而是**不知道在不同平台应该发什么内容、用什么形式**
- 传播效果最好的是短视频和 GIF，但**制作门槛最低且刚需最大的是标注截图和平台文案**
- 没有一个工具能做到"输入产品信息 → 一键生成适合各平台的全套宣传材料"

### 一句话价值主张

**"告诉 DemoGen 你的产品是什么，它帮你在每个平台都发出专业的推广内容。"**

### 目标用户

| 用户 | 典型场景 | 他们需要什么 |
|---|---|---|
| 课程项目团队 | 答辩展示 | 讲稿 + PPT + 演示截图 |
| 独立开发者 | Product Hunt 发布 | Landing Page 文案 + Demo GIF + 社媒帖子 |
| 小程序/App 团队 | 小红书/抖音推广 | 标注截图 + 短视频 + 种草文案 |
| 开源维护者 | GitHub 推广 | README 增强 + 功能 GIF + Twitter 推文 |
| 学长的小工具 | 朋友圈/即刻传播 | 一句话介绍 + 功能亮点图 + 分享链接 |

## 三、核心用户流程（升级后）

```
1. 用户打开 DemoGen，Agent 引导式对话

2. 输入产品信息（任选一种或多种）：
   - 粘贴 GitHub 链接
   - 上传截图
   - 文字描述产品
   - 上传文档

3. AI 分析项目 → 生成"项目理解卡片"（已有功能）

4. 选择目标渠道：
   "你想在哪里推广？"
   □ Product Hunt  □ 小红书  □ Twitter/X
   □ GitHub        □ 抖音/B站  □ 微信/即刻

5. AI 生成"宣传策略"（已有功能，需扩展）：
   - 根据渠道推荐最合适的资产组合
   - 给出内容风格建议（每个平台不同）

6. 用户确认要生成的资产

7. AI 批量生成资产 → 右侧实时预览：
   ┌─────────────────────────────┐
   │ [文案] [截图] [GIF] [讲稿]  │ ← Tab 切换
   │                             │
   │  小红书文案                  │
   │  ──────────                 │
   │  🔥 还在手动整理项目文档？    │
   │  这个小工具帮你一键生成...    │
   │                             │
   │  [编辑] [复制] [换一版]      │
   └─────────────────────────────┘

8. 用户可以：
   - 直接复制文案
   - 下载标注截图
   - 编辑后重新生成
   - 导出全套资产包（ZIP）
```

### Phase 6 详细：多渠道文案生成

新增 CopyWriter subagent：
- 输入：ProjectUnderstanding + 目标渠道列表
- 输出：每个渠道的适配文案（标题、正文、标签/话题）

渠道风格差异：
- **小红书**：种草风，emoji 多，问句开头，标签话题
- **Twitter/X**：精炼，140 字以内，英文 hashtag
- **Product Hunt**：Tagline + 问题 + 解决方案，英文
- **GitHub README**：技术导向，Badge + 功能列表 + Quick Start
- **即刻/V2EX**：技术圈口吻，直接说做了什么，不过度营销

状态机扩展：在 `awaiting_assets` 之后新增 `generating_copy` 状态。

### Phase 7 详细：截图标注

用户上传产品截图 → AI 识别截图中的关键区域 → 自动添加：
- 编号标注（①②③）
- 箭头指向关键功能
- 高亮框
- 简短说明文字

技术方案：Sharp（Node.js 图片处理库）在 Worker 端渲染。

## 八、差异化总结

| 维度 | ChatGPT | Canva | 剪映 | **DemoGen** |
|---|---|---|---|---|
| 理解产品 | ❌ 需要你解释 | ❌ 不理解 | ❌ 不理解 | ✅ 分析 GitHub/截图 |
| 多渠道适配 | ❌ 需要你指定 | ⚠️ 有模板但通用 | ❌ 不涉及 | ✅ 自动适配各平台风格 |
| 文案生成 | ✅ 能做 | ❌ 不做 | ❌ 不做 | ✅ 能做 |
| 图片标注 | ❌ 不能 | ✅ 需手动 | ❌ 不涉及 | ✅ AI 自动标注 |
| GIF/视频 | ❌ 不能 | ⚠️ 简单模板 | ✅ 手动剪辑 | ✅ AI 辅助生成 |
| 开发者专属 | ❌ 通用 | ❌ 通用 | ❌ 通用 | ✅ 专为开发者设计 |

**DemoGen 的核心差异：它是唯一一个"理解你的产品 + 知道每个平台怎么发 + 自动生成全套资产"的工具。**
