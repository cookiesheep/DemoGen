# DemoGen 技术栈独立调研提示词

## 背景

我们是 5 人大二计算机学生团队，在做软件工程课程大作业。项目叫 DemoGen——一个"项目展示资产编排器"：用户输入 GitHub 仓库/README/截图/已部署 URL，系统通过 AI 理解项目，按展示场景（课程答辩/作品集/社媒）生成讲稿、PPT、一页式介绍等资产包；如果有可访问 URL 则额外用 Playwright 录屏。

核心交互模式：不是传统表单填写，而是 **agent 引导式**——左侧 AI 对话区引导用户，右侧实时预览生成的资产，用户可以对话修改也可以直接编辑。

## 团队能力

- 5 人，全部大二，计算机专业
- 有非常浅的前后端经验（大概知道 HTML/CSS/JS，写过简单页面）
- Python 会一点点
- 没有 TypeScript 实战经验，但可以学
- 没有用过任何 AI/Agent 框架
- 没有用过 Next.js / React，但有人了解过
- 开发周期大概 2-3 个月

## 我需要你调研的内容

### 第一部分：全栈框架选型

请对以下方案做**真实对比**，不要只推荐你最熟悉的：

1. **Next.js + Vercel AI SDK + Vercel 部署**
2. **Next.js + CopilotKit + Vercel 部署**
3. **Nuxt.js / Vue 全栈方案**（如果团队更熟悉 Vue）
4. **Python 后端（FastAPI/Flask）+ 简单前端（Streamlit / Gradio / 纯 HTML）**
5. **Dify / Flowise 等低代码 AI 平台 + 自定义前端**
6. **其他你认为更合适的方案**

对每个方案请评估：
- 学习曲线（对 5 个零基础大二学生）
- 开发效率（2-3 个月能做到什么程度）
- AI 集成难度（流式输出、工具调用、agent 引导式交互）
- 部署难度
- 社区资源和文档质量
- 它在 2026 年是否仍然是主流/活跃的（还是已经过时/被替代）
- 具体缺点和坑

### 第二部分：Agent 框架选型

我们的产品需要一个 AI agent 来引导用户完成"项目理解→策略选择→资产生成"的流程。请对以下框架做真实对比：

1. **Vercel AI SDK（ai-sdk.dev）的 agent 能力**
2. **CopilotKit**（copilotkit.ai）
3. **OpenAI Agents SDK**
4. **Claude Agent SDK / Anthropic Agent SDK**
5. **LangChain / LangGraph**
6. **CrewAI**
7. **Dify**
8. **其他你认为适合的框架**

对每个框架请评估：
- 2026 年 3 月的最新状态（版本号、活跃度、是否还在积极维护）
- 是否适合构建**用户面对的产品**（不是开发者工具，是给普通用户用的产品）
- 是否支持"Chat + Canvas"交互模式（对话区 + 实时预览区）
- 流式输出支持
- 工具调用 / Function calling 支持
- 与前端框架的集成程度
- 学习曲线
- 真实的缺点和局限

### 第三部分：关键子系统技术选型

1. **PPT/演示文稿生成**：
   - Marp vs Slidev vs reveal.js vs python-pptx vs Presenton vs Anthropic PPTX skill vs PPTAgent
   - 哪个最适合"AI 生成内容 → 渲染成专业演示文稿"的场景？
   - 哪个能生成最好看的输出？

2. **Playwright 录屏**：
   - Node.js 版 vs Python 版，对我们哪个更合适？
   - 真实的难点是什么？（不是"很简单 3 行代码"这种说法，而是实际部署和使用中会遇到什么坑）

3. **GitHub 仓库解析**：
   - 怎么获取和解析一个 GitHub 仓库的 README、目录结构、关键文件？
   - 用 GitHub API 还是直接 clone？
   - token 和速率限制问题

4. **LLM Provider 选择**：
   - OpenAI vs Anthropic vs 其他（DeepSeek 等）
   - 成本对比（每次生成一个完整资产包大概要多少钱）
   - 哪个在"结构化输出"方面表现最好
   - 如果我们是中国学生，API 访问是否有地区限制或网络问题

### 第四部分：真实风险评估

请不要给我"一切都很简单"的说法。请告诉我：

1. 以我们团队的能力，2-3 个月最现实能做到什么程度？
2. 哪些功能看起来简单但实际上会吃掉大量时间？
3. Agent 引导式 UX 的实现难度到底有多大？一个从未用过相关框架的团队能在 2-3 个月内做出来吗？
4. 你见过的类似学生项目，最常死在什么地方？
5. 如果只能选 3 个核心功能做到极致，你建议选哪 3 个？

### 第五部分：最终推荐

基于以上调研，请给出：
1. **你最推荐的完整技术栈**（前端 + 后端 + AI 框架 + 部署 + 数据库 + 关键工具）
2. **你最推荐的备选方案**（如果推荐方案学不动/遇到阻碍时的降级方案）
3. **按周的学习和开发计划**（2-3 个月，5 人分工）
4. **每个方案的"最小可演示版本"需要多长时间**
